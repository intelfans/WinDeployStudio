WinDeploy Studio 隐私政策

WinDeploy Studio Privacy Policy

生效日期：2026 年 7 月 23 日

Effective date: July 23, 2026

1. 我们收集的信息

WinDeploy Studio（Microsoft Store 版）由 Bob Steve 提供。本应用不要求创建账户，也不会主动收集、上传、出售或共享个人信息。

Store 版不包含广告、遥测、行为分析、崩溃上报或远程 AI 服务。应用设置、日志、磁盘测试历史和导出的报告仅保存在用户自己的 Windows 设备上。

本地记录可能包含用户主动操作产生的磁盘型号、容量、序列号、接口信息、测试结果、文件路径和时间。除非用户主动导出或分享，这些信息不会离开设备。

2. 磁盘操作

应用需要管理员权限才能执行磁盘检测、分区、格式化、安装介质创建和 Windows To Go 等操作。破坏性操作会在应用中提示，并要求用户确认。

应用不会自动上传磁盘内容、ISO 内容或个人文件。

3. 外部网站

当用户主动点击 Microsoft 官方下载页、GitHub 反馈、SourceForge 仓库或其他外部链接时，系统浏览器会打开相应网站。此后产生的数据由对应网站按照其自身隐私政策处理，WinDeploy Studio 不控制这些网站。

4. 信息共享

WinDeploy Studio 不会将本地日志、磁盘信息、测试记录或文件内容出售、出租或自动共享给第三方。

5. 本地数据保留与删除

应用日志和测试历史保存在本地应用数据目录中。日志会按应用设置自动清理；用户也可以在应用中删除测试记录和日志，或通过 Windows 删除应用数据。

卸载应用不会保证自动删除用户主动生成的导出文件。

6. 安全

应用使用 Windows 文件权限和管理员权限机制保护本地数据。用户应避免将包含磁盘序列号、路径或测试信息的报告公开分享。

7. 政策变更

如果应用的数据处理方式发生变化，我们会更新本页面的生效日期和内容。

8. 联系方式

如需反馈，请访问 GitHub Issues。请不要在公开 Issue 中提交 API 密钥、密码、序列号或其他敏感信息。

1. Information we collect

WinDeploy Studio (Microsoft Store edition) is provided by Bob Steve. The application does not require an account and does not actively collect, upload, sell, or share personal information.

The Store edition contains no advertising, telemetry, analytics, crash reporting, or remote AI service. Application settings, logs, disk benchmark history, and exported reports are stored locally on the user's Windows device.

Local records may contain disk model, capacity, serial number, interface information, benchmark results, file paths, and timestamps created by the user's actions. These records do not leave the device unless the user explicitly exports or shares them.

2. Disk operations

The application requires administrator permission for disk inspection, partitioning, formatting, installation-media creation, and Windows To Go operations. Destructive operations are explained in the application and require user confirmation.

The application does not automatically upload disk contents, ISO contents, or personal files.

3. External websites

When the user opens a Microsoft download page, GitHub feedback page, SourceForge repository, or another external link, the system browser opens that website. The website then processes information under its own privacy policy; WinDeploy Studio does not control those websites.

4. Sharing

WinDeploy Studio does not sell, rent, or automatically share local logs, disk information, benchmark records, or file contents with third parties.

5. Local retention and deletion

Application logs and benchmark history are stored in the local application-data directory. Logs are cleaned according to the application settings. Users can delete benchmark records and logs in the application or remove the application data through Windows.

Uninstalling the application does not guarantee removal of files that the user explicitly exported.

6. Security

The application relies on Windows file permissions and administrator controls to protect local data. Avoid publicly sharing reports that contain disk serial numbers, paths, or benchmark details.

7. Changes to this policy

If the application's data practices change, we will update this page and its effective date.

8. Contact

For feedback, visit GitHub Issues. Do not post API keys, passwords, serial numbers, or other sensitive information in public issues.
